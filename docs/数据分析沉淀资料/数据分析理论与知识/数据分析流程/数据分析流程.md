# 数据分析流程

**CRISP-DM**全称是**Cross-Industry Standard Process for Data Mining**，翻译为**跨行业数据挖掘标准流程**。
它是目前数据分析与数据挖掘领域里应用最广泛的流程框架之一，帮助分析人员规范化地开展数据项目。

CRISP-DM 六个阶段，业务理解 (Business Understanding)，数据理解 (Data Understanding)，数据准备 (Data Preparation)，建模 (Modeling)，评估 (Evaluation)，部署 (Deployment)

除了**CRISP-DM** 之外，数据分析和数据挖掘领域还有不少流程框架，不同的行业、团队、项目类型会选择不同的流程模型。下面我给你整理一个对比表，方便你了解它们的特点：

***

## **常见数据分析 / 数据挖掘流程框架**

| 流程框架          | 全称                                              | 应用领域          | 核心阶段                                  | 特点                   |
| ------------- | ----------------------------------------------- | ------------- | ------------------------------------- | -------------------- |
| **CRISP-DM**​ | Cross-Industry Standard Process for Data Mining | 通用数据挖掘        | 业务理解 → 数据理解 → 数据准备 → 建模 → 评估 → 部署     | 业界最常用，标准化程度高，兼容不同算法  |
| **KDD**​      | Knowledge Discovery in Databases                | 学术研究、挖掘项目     | 数据选择 → 数据预处理 → 数据挖掘 → 模式评估 → 知识表示     | 强调知识发现过程，早期经典方法      |
| **SEMMA**​    | Sample, Explore, Modify, Model, Assess          | SAS 数据挖掘      | 采样 → 探索 → 修改 → 建模 → 评估                | SAS 推出的闭环模型，更偏技术实现   |
| **OSEM**​     | Open Standard for Data Mining                   | 企业数据挖掘        | 定义问题 → 获取数据 → 准备数据 → 数据挖掘 → 结果解释 → 应用 | 强调数据挖掘与业务结合          |
| **DMAIC**​    | Define, Measure, Analyze, Improve, Control      | 六西格玛、制造、流程优化  | 定义 → 测量 → 分析 → 改进 → 控制                | 管理和工程领域常用，偏质量改进和流程优化 |
| **PDCA**​     | Plan, Do, Check, Act                            | 制造业、管理、数据驱动改进 | 计划 → 执行 → 检查 → 行动                     | 持续改进循环，可延伸到数据分析项目    |
| **OODA**​     | Observe, Orient, Decide, Act                    | 决策科学、商业分析     | 观察 → 定位 → 决策 → 行动                     | 偏快速决策与实时数据分析         |

***

## **选择框架的建议**

- **业务导向强** → CRISP-DM、DMAIC
- **数据科学/学术研究** → KDD
- **使用 SAS 平台** → SEMMA
- **需要持续改进** → PDCA、DMAIC
- **实时分析决策** → OODA
