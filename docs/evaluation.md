# Datai 评测方案（阶段 1）

更新时间：2026-02-19

## 1. 评测目标
验证 MVP 是否达到“可用且可解释”，避免做出只能演示的玩具系统。

## 2. 评测对象
- 会话级分析结果（结论、建议、图表）。
- 执行过程可观测性（任务状态面板事件）。

## 3. 指标体系

### 3.1 用户价值指标
- 相关性：是否符合用户分析意图。
- 有用性：输出建议是否具备业务价值。
- 清晰度：表达是否简洁易懂。
- 可执行性：用户是否可以据此直接行动。
- 新颖性：是否提供了非模板化的有效洞察。

### 3.2 过程质量指标
- 动作信息完整性：必要步骤是否记录完整。
- 动作顺序可追溯性：事件顺序是否可读可核对。
- 对象细节准确性：字段、筛选条件、指标名称是否准确。
- 错误可诊断性：失败步骤是否有清晰提示。

## 4. 评分方式（建议）
- 单项 0-2 分，单次任务总分 20 分。
- 通过线：16 分。
- 关键指标门槛：相关性、可执行性任一低于 1 分则直接不通过。

## 5. 数据集建议
- 阶段 1 使用经典公开表格数据集做回归评测（如 Titanic、House Prices）。
